# 궤도별·날짜별 수체 면적 (pixel_perrow 기준)

`build_water_per_date_otsu.py`가 만든 궤도별·날짜별 Otsu 수체 지도
(`downloads/water_otsu/flood_water_total_<날짜>_o<궤도>.tif`)의 면적을
`water_area_report.py`로 산정한 결과다. 면적 산출 방식(픽셀 vs 폴리곤)과
정확도 논의는 [FLOOD_DETECTION_KR.md](FLOOD_DETECTION_KR.md) 8-E절 참고.

## 채택 값: pixel_perrow (행별 위도 보정)

이 표의 면적은 **pixel_perrow** — 수체(=1) 화소를 세되, EPSG:4326에서 경도
길이가 `cos(위도)`로 줄어드는 것을 **행(위도)마다 반영**한 값이다. 장면 중심
위도 하나로 근사하는 기존 방식(pixel_centerlat)보다 정확하며, 폴리곤 측지
면적과 소수점까지 일치함을 검증했다(8-E절). 정밀 면적이 필요할 때 이 값을
쓴다.

원자료: `downloads/water_otsu/water_area_perrow.csv`
(컬럼: file, n_water_px, n_valid_px, area_pixel_centerlat_km2, area_pixel_perrow_km2).

## 주의 — 이 면적은 "상태 면적"이지 "침수 면적"이 아니다

이 수치는 각 관측 시점에 **dB가 물처럼 어두운 전체 화소**(서해·큰 강·저수지·
상시 수체 포함)의 면적이다. baseline 대비 늘어난 **신규 침수량**이 아니므로,
홍수 피해 규모를 말할 때는 baseline 비교 기반 산출물(`flood_water_strict`,
[FLOOD_DETECTION_KR.md](FLOOD_DETECTION_KR.md) 2·4절)을 써야 한다. 또한 7/16
o003704는 Otsu 실패로 fallback(-16 dB)이 적용된 값이라 다른 궤도와 임계값
기준이 다르다(8-C절).

## 재현

```bash
# 전체(픽셀 2방식) -> CSV
conda run -n s1_snappy python water_area_report.py --csv downloads/water_otsu/water_area_perrow.csv

# 특정 파일 + 폴리곤 측지 교차검증
conda run -n s1_snappy python water_area_report.py \
    --water downloads/water_otsu/flood_water_total_20260703_o008384.tif --polygon
```
